There are various ways in which subset sum problem is asked in interviews. Here are some of them -

Finding a subset of numbers whose sum is exactly equal to a given number:
 O(n^2) – similar to Knapsack problem

Maximum Value contiguous subsequence sum:
 O(n) using dynamic programming

Finding a pair of number whose sum is equal to a given number: O(n log n)
          a)      sort the numbers and then perform binary search to find other number (sum – first number) in the array
           a.2) Instead of performing binary search on sorted numbers, we can start from the begin index and end index,  and keep on checking whether sum of numbers at the index is <, > or =. If its = , then done else if its < then forward begin_index to right else move end_index to left.  

          b)      O(n) using a hashmap, add all the numbers to the map and check if (sum – first number) is in the map

I am not providing implementations for first two, but in case someone needs it, post your comment below and I will add it here.
