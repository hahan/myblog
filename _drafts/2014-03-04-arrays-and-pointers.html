Let say, you have:
char b[] = "abc";
What is the difference when you say -
char a[] = b;
char *p = b;

To understand the difference, lets see how the compiler behaves when it sees both the declarations. Whenever a compiler reads a variable, it stores it in its symbol table - its names, address etc along with other details. For simplicity, lets assume it stores: [variable name - its address] only.

In the above example, lets say value "abc" is stored at address 1000.

In second case, the compiler will store in its symbol table: a - 1000, where 1000 is address where "abc" is stored. So at runtime, the process will go and fetch the value at address 1000.

But in first case, the compiler will store: p - 2000 (which is the address of the variable p) in its symbol table. That is, the pointer is assigned a different memory address where it can hold the address (which is of array a).

Now at runtime, we need to go to memory 2000, fetch value at that address which is 1000 and now again go to 1000 and fetch the value - this time it will of the array element. - Hence indirection.
