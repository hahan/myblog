Generating a given word through series of one character changes in another word.
eg. fork->ford->word->work

To achieve this, we need to use concept of graphs where each node will represent a word, and the edge will represent one letter change between the 2 words. Lets assume that there is a function called isValidWord(string); which checks whether the word is present in the dictionary.

I am giving below my version of the algorithm, please comment if you think it can be improved further.
    1. First start with given word, create a node for it and add it to a queue.
    2. While the queue is not empty
           2.a) Remove the node N from the queue;
           2.b) Add all the valid neighbour nodes to that node - i.e. find all the words with one character different from the words and create a node for each such word and      add it as children node for the node N. In this step we need to make sure that if we get a word which turns out to the destination word, then we may not add all other nodes as the children node. And also, while adding a node, we need to make sure that that word is not already present in the graph somewhere else. As if that word is already present somewhere then anyhow we will use it to get to the final word.
           2.c) Now, add all those children nodes to the queue.
    3. Now once we got the graph, perform a breadth first search to get to the final word - this would be the shortest path.
